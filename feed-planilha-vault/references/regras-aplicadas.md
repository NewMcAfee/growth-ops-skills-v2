# Regras Aplicadas — feed-planilha-vault

Práticas validadas (pesquisa + caso empírico Sigo ERP 2026-06-26) operacionalizadas como regras concretas. Cada uma é invariante da skill, não sugestão.

| # | Prática validada | Evidência / fonte | Regra concreta na skill |
|---|---|---|---|
| R1 | Export CSV por **nome de aba** via `gviz/tq?tqx=out:csv&sheet={nome}` | Ativo set/2025; 86,9% mais rápido que Apps Script ([Medium/Abu Abdirohman](https://medium.com/@abuabdirohman/google-sheets-integration-why-csv-beats-google-apps-script-by-86-9-ae07c0b17f1d)) | `feed-download.ps1` usa esse endpoint. Dispensa descobrir `gid`. Nome URL-encoded (`[uri]::EscapeDataString`) p/ suportar espaço/acento. |
| R2 | Fallback `/export?format=csv&gid={gid}` quando gviz distorce tipos | gviz passa pela Visualization API e pode inferir/reformatar tipos por locale ([metricrat-ai2](https://sites.google.com/view/metricrat-ai2/guides/use-gviz-to-get-and-query-google-sheet-data)) | Se QA pós-download acusar distorção sistemática que não existe na planilha, trocar p/ endpoint `/export` com `gid` (documentado em troubleshooting.md). |
| R3 | Tarefa agendada robusta a desligamento **e bateria** | `-StartWhenAvailable` cobre missed run; `-AllowStartIfOnBatteries`/`-DontStopIfGoingOnBatteries` cobrem laptop ([MS Learn New-ScheduledTaskSettingsSet](https://learn.microsoft.com/en-us/powershell/module/scheduledtasks/new-scheduledtasksettingsset)) | `register-task.ps1` aplica as 3 flags + `-RestartCount 3 -RestartInterval 5min`. Sem `AllowStartIfOnBatteries`, num laptop a tarefa **não dispara na bateria** (default Windows). |
| R4 | `.ps1` deve ser **UTF-8 com BOM** no PowerShell 5.1 | Bug real Sigo ERP: PS 5.1 lê `.ps1` sem BOM como ANSI → trava em `—`/acentos | Scripts versionados com BOM e **copiados byte-a-byte** (`Copy-Item`), nunca escritos/editados pelo Claude na pasta-destino. O que varia vai em `feed-config.json` (JSON tolera sem BOM). |
| R5 | Sobrescrita **atômica** com validação prévia | "bad data is worse than no data" (Avo; princípio da skill `newton`) | Baixa `.tmp` → valida (tamanho ≥ minBytes + header contém âncora + linhas ≥ minLineRatio×atual) → só então `Move-Item -Force`. Falha → mantém dado bom, loga `WARN`. |
| R6 | **PII nunca em claro** no Git | Princípio `newton`; LGPD | `scan-fonte.py` detecta PII (cabeçalho + regex email/cpf/cnpj). CSV com PII → entra no `.gitignore` antes da 1ª carga. Agregado (gasto/MQL/CAC) é commitável. |
| R7 | QA de qualidade na origem | Caso Sigo ERP: 13 colunas formatadas como Porcentagem corromperam phone/datas/ids; ID longo como Número virou notação científica | `scan-fonte.py` reporta colunas contaminadas por nome de cabeçalho + formato-alvo (Texto/Data) pra correção **na planilha**, não no CSV (correção no CSV é sobrescrita no próximo run). |
| R8 | Mecanismo **local**, não cloud | Cloud agent não vê filesystem local (vault OneDrive); MCP Google Drive some em runs headless | Sempre PowerShell + Agendador local. A skill recusa propor `/schedule` cloud ou loop do Claude pra esse fim. |
| R9 | Download via **`curl.exe`**, não `System.Net.WebClient` | Caso Sigo ERP: `WebClient.DownloadFile` tem timeout oculto de **100s** que estourava em aba grande (CRM 8MB+, gviz lento server-side); `curl` baixou o mesmo arquivo em ~3s | `feed-download.ps1` usa `curl.exe -sL --fail --max-time 600 -o`. Streaming pra disco (sem teto de memória), `--fail` transforma HTTP≥400 em erro, preserva UTF-8 cru. `curl.exe` é nativo no Windows 10 1803+. |
| R10 | Preferir endpoint **`/export?format=csv&gid=`**, não gviz, p/ abas com fórmula | Caso Sigo ERP 2026-06-27: aba `leads_pipeline` (montada com QUERY/IMPORTRANGE) veio **truncada a 49 de 6.193 registros** pelo gviz, que materializa só o range visível; `/export?gid=` trouxe os 6.193. Abas de valores estáticos não sofrem. | Config usa `gid` por aba (não nome). Motor **tenta** `/export?format=csv&gid={gid}` e, se o endpoint falhar (ex: HTTP 401/403 — ver R11), **cai automaticamente pra `gviz?gid`** (mesmo gid, sem truncar quando não há fórmula). **Gids** obtidos parseando `.../htmlview` (JSON `items.push({name,gid})`) ou do `#gid=` na URL da aba aberta. Validar pós-1ª-carga com contagem de registros esperada (o truncamento é silencioso — header e tipos vêm certos, só faltam linhas). |
| R11 | **Fallback automático `/export` → gviz** quando a org bloqueia o export anônimo | Caso Grupo SA 2026-06-29 (growthpack V4): `/export?gid` devolve **HTTP 401 anônimo** mesmo com a planilha "qualquer pessoa com o link: Leitor" — política do Google Workspace restringe o endpoint `/export`; `htmlview` e `gviz` continuam 200. | Motor monta uma **cadeia de candidatos** por target (`export?gid` → `gviz?gid`; ou `gviz?sheet` sem gid) e baixa do primeiro que responder 200, logando `fallback automático` quando não é o preferencial. Operador **não precisa** remover o gid do config — mantém R10 (export quando disponível) e degrada pra gviz sozinho. `curl.exe -w '%{http_code}'` captura o status pro log. |

## Síntese dupla

A skill **INCORPORA** o export gviz validado (R1), o padrão de 4 flags de Task Scheduler robusto a laptop/desligamento (R3), a sobrescrita atômica com guarda (R5) e o princípio PII-no-gitignore (R6) — **E PREENCHE** a lacuna que nenhum tutorial de "baixar Sheet como CSV" cobre: o **ciclo completo de governança de um feed dentro de um vault** (detecção de PII → gitignore → QA de qualidade da origem → cascata sináptica de docs → tarefa idempotente), porque o valor não está no download (trivial), está em **fazer o feed nascer auditável e seguro** num repositório versionado com dado de cliente.
