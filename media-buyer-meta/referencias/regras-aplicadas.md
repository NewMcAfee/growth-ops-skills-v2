# Regras Aplicadas — estado da arte Meta Ads 2024-2026 → regras concretas

Práticas validadas (com fonte datada) operacionalizadas como regra. Releia antes de criar/editar estrutura. Números de limites mudam por versão da API — revalide na versão vigente quando houver erro.

| # | Regra | Threshold / ação | Fonte |
|---|---|---|---|
| **R1** | Criar tudo `status=PAUSED`; ativar só após QA | Ads de API passam por review normal (sem bypass). Cheque `approval_status` antes de ativar | [AdManage 2026](https://admanage.ai/blog/meta-ads-api) |
| **R2** | Preferir `image_hash` a `image_url` | Referencie hash pré-existente. MCP não sobe imagem → operador sobe na biblioteca (case por nome) ou URL pública. `object_story_spec {page_id, story_id}` preserva social proof | [AdManage 2026](https://admanage.ai/blog/meta-ads-api) |
| **R3** | Respeitar limites do `asset_feed_spec` | ≤30 assets (≤10 img, ≤10 vídeo), ≤5 bodies, ≤5 titles, ≤5 descriptions, ≤5 CTAs, ≤5 links. body ≤1024 chars; title/desc ≤255. Validar **antes** do POST | [Meta Dev](https://developers.facebook.com/docs/marketing-api/ad-creative/asset-feed-spec/options/) |
| **R4** | Consolidar, não fragmentar | B2C: 3-4 campanhas. Ad set denso (~20-25 criativos diversos) > 5 ad sets fragmentados (+17% conv / -16% custo). Alertar contra splittar audiência | [GrowthMarketer](https://growthmarketer.com/blog/meta-ads-performance-system/), [Tiger Pistol](https://tigerpistol.com/metas-performance-5-framework-the-impact-of-simplifying-your-account-structure/) |
| **R5** | ABO testa → CBO escala | CBO default (Meta realoca budget). ABO quando ad sets têm tamanhos muito díspares ou exige controle por audiência | [SuperScale](https://superscale.ai/learn/cbo-vs-abo-advantage-plus/) |
| **R6** | Mirar 50 conversões/ad set/7 dias | Dimensionar budget+audiência pra cruzar o limiar de saída da learning phase (signal density). Alertar se a combinação budget×audiência torna isso inviável | [Tiger Pistol](https://tigerpistol.com/metas-performance-5-framework-the-impact-of-simplifying-your-account-structure/) |
| **R7** | Evitar edição que reseta learning | Resetam: budget Δ>20-25%, mudança de audiência, de evento de otimização, de bid strategy, swap de criativo. Acima do limiar → **duplicar** ad set em vez de editar | [HyperFX](https://www.hyperfx.ai/blog/meta-ads-learning-phase-stuck) |
| **R8** | Prevenir audience overlap/self-competition | Exclusões waterfall entre ad sets de temperaturas diferentes (`waterfall-exclusions.md`). Overlap splita atribuição e impede cruzar as 50 conv | [FiveNine](https://fiveninestrategy.com/meta-campaign-structure-over-retargeting/) |
| **R9** | Declarar `special_ad_categories` | Housing/credit/employment/financial/issues. **Saúde-farmácia** = atenção redobrada (políticas de saúde). SAC força idade 18-65+, todos gêneros, sem ZIP, sem Lookalike. Setor regulado → criativo **manual**, não Advantage+ full-auto | [Data Axle](https://www.data-axle.com/resources/blog/meta-special-ad-categories-rules/), [Birch](https://bir.ch/blog/meta-advantage-plus-guide) |
| **R10** | Vertical-first + AI-disclosure | 9:16 primário (≈90% do inventário é vertical), 4:5 secundário p/ feed. Desde mar/2026 Meta exige **disclosure de conteúdo gerado/modificado por IA** — causa comum de rejeição | [AdMove 2026](https://www.admove.ai/blog/meta-advantage-creative-best-practices-for-2026) |
| **R11** | Tratar rate limit por BUC | write=3 pts, read=1 pt. Em erros 613/80004 → backoff (não loop cego). Diagnosticar via `ads_get_errors`. Erros 100/200 = permissão (scope/role) | [Meta Dev](https://developers.facebook.com/docs/marketing-api/overview/rate-limiting/) |
| **R12** | Naming enforçado antes de tocar a conta | Validar nome campo a campo contra a taxonomia ANTES do POST (previne ~90% do drift). Nome do anúncio carrega versão/variação p/ leitura de winners. Nunca duplicar IDs no nome | [AdManage Naming](https://admanage.ai/blog/facebook-ad-naming-convention), [Bestever](https://www.bestever.ai/post/ad-naming-conventions) |
| **R13** | Evento de fundo escasso → NÃO otimizar direto | Otimizar por evento profundo e raro (ex: Demo Agendada/Realizada) com **< ~50 conv/ad set/semana** (R6) trava o learning permanente e PIORA a entrega — mesmo sendo o evento mais alinhado ao negócio. Antes de mirar fundo: checar volume → consolidar ad sets pra concentrar sinal → se faltar volume, usar **value-based** (otimiza no evento abundante MQL/Lead com valor maior p/ quem avança no funil; captura qualidade sem depender do evento raro). Trocar o evento NÃO resolve cold-start sozinho (o desvio acontece antes da 1ª conversão) | [campo Sigo ERP 2026-06] |
| **R14** | Proteger o vídeo no cold-start de criativo | Em CBO, antes da 1ª conversão a Meta distribui pelo sinal raso (CTR/engajamento), o **estático ganha o early-signal e o vídeo não recebe budget** pra provar conversão. Proteger via: (a) formato em **campanha** própria (budget isolado), (b) **mínimo diário por ad set** (CBO + minimum spend) forçando o start, ou (c) vídeo de topo otimizando Lead/ThruPlay (evento abundante, sem cold-start) alimentando o retargeting. Broad + Advantage+ Audience: **remover o "narrowing"** (camadas AND "e também devem corresponder a", estreitam demais) e manter só a 1ª camada de interesse como **sugestão** (não filtro — o painel expande além) | [campo Sigo ERP 2026-06] |

## Macros de URL da Meta (8 oficiais)

Usar em `url_tags` ou no `link_url`: `{{ad.id}}`, `{{ad.name}}`, `{{adset.id}}`, `{{adset.name}}`, `{{campaign.id}}`, `{{campaign.name}}`, `{{placement}}`, `{{site_source_name}}`. Funcionam em parâmetros de URL — não em deep links arbitrários. ([AdWhatCost](https://adwhatcost.com/blog/complete-guide-on-how-to-use-utm-parameters-and-url-macros-to-track-facebook-ads.html))

## Contexto 2024-2026 (datado)

- **Andromeda** (ranking, ~2024-25): processa milhares de variantes em paralelo → ganho migrou de splittar audiência p/ empilhar diversidade criativa no ad set consolidado. Criar 15-25 ads *genuinamente diferentes*, não 15 versões do mesmo.
- **Advantage+ Shopping → "Advantage+ Sales"** (rename início 2025), expandido p/ lead gen e app.
- **Legacy Advantage Shopping API sunset Q1 2026 (v25)** — pinar a versão da API.
- **Tokens:** preferir System User token (renovável) p/ automação; scopes `ads_management` + `ads_read` + `business_management`.
